# TenderPojectPublick
VBA Tender Manager – A VBA-based Excel project designed to store and manage tender information and other related business data. Features include a user-friendly interface with UserForms, data search and filtering capabilities, and automation of routine table operations.

**Project "Current Tenders 2.0"**
The Current Tenders 2.0 project is designed for convenient display and editing of tender information. All data is stored in rows of a table located on the first sheet of the document.
Main Features
**Search Bar**
A search bar is provided on the main page. When entering keywords, only rows containing those words are displayed.
 
![image](https://github.com/user-attachments/assets/b5d26236-9556-4e44-b1ff-6604639d5966)

**Displaying Active Tenders**
By selecting the corresponding radio button, only tenders with a valid document submission deadline are displayed. The data is sorted from the nearest to the farthest submission deadlines.
![image](https://github.com/user-attachments/assets/5e789daf-a29f-4e54-8396-cd452e02f44c)
**Tender Statuses**
Each tender has a status displayed in the table. Status names are stored on the second sheet and can be edited as desired.

Due to the large number of columns, the table may not be very convenient for viewing. Horizontal scrolling can cause the cursor to accidentally jump to another row. For improved clarity, a Tender Passport Form is provided.

**Tender Passport Form**
When double-clicking a row in the table, the tender passport opens — a special form (UserForm) that displays information gathered from the relevant columns.
![image](https://github.com/user-attachments/assets/1654c206-acc8-49c5-87f8-9da3e594b4bc)

**Expanded Tender Passport Form**
By clicking the button, the expanded form of the tender passport will be displayed.
![image](https://github.com/user-attachments/assets/13cc0fe6-2741-4cd4-a59f-2f02b436c89c)
The expanded form will show fields with files and links, including a link to the tender's website and a CRM link. It also displays files stored in the designated folder.
**Working with Links**
After filling in these fields, the “CRM” and “Link” buttons become active, which open the corresponding links in the Chrome browser. If Chrome is not installed, this function will not work.
**Working with Folders**
The form allows you to create and open a folder for storing tender-related documents.
•	Creating a Folder: If the folder hasn't been created yet, a button to create it will appear in the form. After pressing the button, a window will appear to select the directory. 
![image](https://github.com/user-attachments/assets/a6a28c85-b2cc-4903-91fa-2a4289d64e19)
The folder name will be automatically generated based on the "Tender Number" and "Client Name" fields.
![image](https://github.com/user-attachments/assets/bd9b5f8e-81ad-4552-b0ad-e64f6976b23f)
Once the folder is created, the button will display as "Open Folder." By pressing it, the created folder will open.

**Changing the Default Path**
To change the default path, open the VBA editor (Alt + F11), find the ConstantsTableColumn module, and specify a new directory path. This path will be used for creating folders by default.
![image](https://github.com/user-attachments/assets/184b451d-6372-4342-b8ff-7f1258a31f85)
Saving the Path
The path to the created folder is saved in the 25th column of the table. If necessary, you can manually change this path. You can also manually insert any folder name into the 25th column if desired.
Folder with Statutory Documents
The form also includes a button for opening a folder with the company’s statutory documents. This can be useful when preparing a tender application to avoid searching for the necessary folder. Setting up this folder is done in the same way as setting the default folder for tenders, except you need to specify the path in the DirectoryStatutoryDocuments variable.
![image](https://github.com/user-attachments/assets/abf7c5d8-da52-434f-b0ae-a031706c3b31)
